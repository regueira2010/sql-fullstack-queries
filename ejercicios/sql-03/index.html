Ejercicio 02: Finanzas Personales

Análisis de deudas y cobros mediante el uso de funciones de agregación, cálculos aritméticos y optimización de flujos de pago.

PostgreSQL 18.3 Análisis DQL Agregaciones SUM/AVG

Paso 1: Control de Registros

Se define la tabla principal para centralizar acreedores y deudores.

CREATE TABLE finanzas_personales (
    nombre VARCHAR(20) PRIMARY KEY,
    me_debe INTEGER,
    le_debo INTEGER,
    cuotas_pagar INTEGER
);

La Primary Key asegura que no existan registros duplicados para una misma entidad o persona.

Reportes de Inteligencia Financiera

1. Máximo Acreedor

SELECT nombre, le_debo FROM finanzas_personales 
ORDER BY le_debo DESC LIMIT 1;

Identificación de la entidad o persona a la que se le adeuda la mayor cantidad de capital.

2. Máximo Deudor

SELECT nombre, me_debe FROM finanzas_personales 
ORDER BY me_debe DESC LIMIT 1;

Determinación del mayor flujo de ingreso pendiente por cobrar.

3. Deuda Total

SELECT SUM(le_debo) AS total_deuda FROM finanzas_personales;

Uso de SUM() para calcular el total acumulado de la deuda.

4. Deuda Promedio

SELECT AVG(le_debo) AS promedio_deuda 
FROM finanzas_personales WHERE le_debo > 0;

Cálculo del promedio de deuda utilizando AVG(), filtrando registros sin deuda activa.

5. Proyección de Tiempo (Años/Meses)

SELECT SUM(cuotas_pagar) AS total_cuotas,
       SUM(cuotas_pagar) / 12 AS años,
       SUM(cuotas_pagar) % 12 AS meses
FROM finanzas_personales;

Desglose del tiempo total de pago mediante división entera y operador módulo.

6. Valor de Cuota tras Cobro Total

SELECT (SUM(le_debo) - SUM(me_debe))/ SUM(cuotas_pagar) AS Valor_Cuota 
FROM finanzas_personales;

Cálculo aritmético para determinar la mensualidad si se aplicaran todos los cobros a la deuda.

7. Registro de Compromiso Inesperado

INSERT INTO finanzas_personales (nombre, me_debe, cuotas_cobrar, le_debo, cuotas_pagar)
VALUES ('pareja', 0, 0, 50000, 1);

Uso de INSERT INTO para actualizar la realidad financiera tras un nuevo compromiso.

8. Gasto Operativo Mensual (Actual)

SELECT SUM(le_debo / cuotas_pagar) AS cuota_mes 
FROM finanzas_personales WHERE cuotas_pagar > 0;

Suma de las parcialidades de todas las deudas activas para este mes.

9. Actualización de Cuotas (Refinanciamiento)

UPDATE finanzas_personales 
SET cuotas_pagar = 13 
WHERE nombre = 'almacén esquina';

Modificación de registros existentes mediante UPDATE para ajustar plazos de pago.

10. Recálculo Final de Cuota

SELECT SUM(le_debo / cuotas_pagar) AS cuota_final 
FROM finanzas_personales WHERE cuotas_pagar > 0;

Resultado final de la carga mensual tras todas las actualizaciones de datos.
